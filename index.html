Git e GitHub

Git: O que é e Como Usar

O Git é um Sistema de Controle de Versão (VCS) distribuído, essencial para desenvolvedores e equipes que desejam rastrear e gerenciar alterações no código de forma eficiente. Com o Git, você pode:

Rastrear Alterações: Manter um registro detalhado de todas as modificações no seu projeto, permitindo reverter facilmente para versões anteriores, se necessário.
Colaboração: Facilitar o trabalho em equipe, permitindo que várias pessoas trabalhem no mesmo projeto simultaneamente e mesclem suas alterações em uma fonte final.
Para começar a usar o Git, você precisará instalar o software e aprender comandos básicos como git init, git add, git commit, git push e git pull. Esses comandos permitem que você inicie um repositório, adicione arquivos, registre alterações e sincronize seu trabalho com um repositório remoto. Neste site você aprenderá um pouco mais sobre como funciona cada comando.

GitHub: O que é e Como Usar

O GitHub é uma plataforma online que utiliza o poder do Git, oferecendo uma interface intuitiva e recursos adicionais para facilitar a colaboração. No GitHub, você pode:

Hospedar Repositórios: Armazenar seus projetos na nuvem, acessíveis de qualquer lugar.
Colaboração em Massa: Trabalhar com desenvolvedores do mundo todo, contribuindo para projetos open source ou privados.
Ferramentas Avançadas: Utilizar uma variedade de ferramentas para automação, integração contínua e gestão de projetos.
Para usar o GitHub, crie uma conta gratuita, crie um repositório e comece a colaborar!
Comandos básicos do Git são utilizados junto com a interface do GitHub para gerenciar projetos de maneira eficiente. Além disso, você pode explorar recursos como issues, pull requests e GitHub Actions para melhorar seu fluxo de trabalho.

Comando Funcionalidade Uso do Comando
init Cria um novo repositório Git. $ git init
clone Clona um repositório Git existente para sua máquina local. $ git clone <url_do_repositorio>
branch Exibe, cria, lista e gerencia branches (ramos) no seu repositório. $ git branch (lista branches)
$ git branch <nome_da_branch> (cria uma nova branch)
checkout Troca para uma branch específica ou recupera arquivos para o seu diretório de trabalho. $ git checkout <nome_da_branch>
status Mostra o status dos arquivos no seu repositório Git (modificados, encenados, etc.). $ git status
diff Mostra a diferença entre versões de arquivos. $ git diff (modificações não encenadas)
$ git diff --cached (modificações encenadas)
add Adiciona arquivos ao staging area (área de preparação) para o próximo commit. $ git add <nome_do_arquivo>
commit Captura o estado atual do seu projeto em um novo commit. $ git commit -m "<mensagem_do_commit>"
push Envia os commits locais para um repositório remoto. $ git push <origem> <branch>
pull Baixa alterações do repositório remoto e incorpora-as ao seu repositório local. $ git pull <origem> <branch>
revert Desfaz um commit específico, criando um novo commit reverso. $ git revert <commit_hash>
merge Combina alterações de duas branches diferentes em uma única branch. $ git merge <branch_para_incorporar>
stash Salva temporariamente alterações não encenadas para que você possa trabalhar em outra coisa. $ git stash (cria stash)
$ git stash pop
rm Remove arquivos do seu repositório Git. $ git rm (nome_do_arquivo)
config Configura variáveis globais ou locais do Git. $ git config --global <chave> <valor> (configuração global)
$ git config <chave> <valor> (configuração local)
reset Move o ponteiro HEAD para um commit específico. $ git reset --hard HEAD~1
remote Exibe, adiciona, remove e gerencia repositórios remotos. $ git remote (lista remotos)
$ git remote add <apelido> <url_do_repositorio> (adiciona um remoto)
fetch Baixa alterações do repositório remoto, mas não as incorpora ao seu projeto local. $ git fetch <origem>
show Mostra o conteúdo de um commit específico. $ git show <commit_hash>
help Mostra ajuda sobre comandos Git específicos. $ git help <comando>
rebase Reapplica uma série de commits em cima de outra branch. $ git rebase [nomeDaBranch]
$ git rebase -i [referência]^
blame Mostra qual autor modificou cada linha de um arquivo pela última vez. $ git blame <arquivo>
tag Cria, lista, exclui e gerencia tags (marcadores) em seu repositório. $ git tag (lista tags)
$ git tag <nome_da_tag> <commit_hash> (cria uma tag);
